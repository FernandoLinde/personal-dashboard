Build a simple personal dashboard web app that tracks recent YouTube videos from a fixed list of channels I provide. This is a private tool for daily personal use, not a public consumer app. Prioritize reliability, speed, clean UX, and low maintenance over visual complexity.

Product goal:
Create a YouTube monitoring dashboard that automatically pulls all videos uploaded in the last 7 days from my selected channels, stores them, shows them in one reverse-chronological feed, lets me search and filter them, generates a short AI summary for each video, and lets me download the transcript for each video.

Core requirements

Dashboard and feed

Show all tracked videos from the last 7 days in one unified feed

Sort by publish date/time descending, newest first

Each video card should include:

thumbnail

title

channel name

category

publish date

button to watch on YouTube

3-4 short bullet point AI summary

button to view transcript

button to download transcript as .txt

transcript source label:

"YouTube transcript"

"Generated transcript"

"Transcript unavailable"

Search and filters

Search only across:

video title

channel name

publish date

Add filters for:

category: Tech / Macro / General

channel

Add a "clear filters" action

Keep the UI very fast and easy to scan

Transcript logic

First try to fetch the native YouTube transcript/captions

If unavailable, fall back to speech-to-text transcription

Store the transcript text in the database

Allow transcript viewing in the UI

Allow direct download as plain text

Clearly show which transcript source was used

AI summary logic

Generate 3-4 concise bullet points for each video

Summaries should be short, scannable, and factual

Avoid fluff, intro language, and generic phrases

Focus on the main discussion topics, arguments, announcements, or takeaways

Store summaries so they are not regenerated on every page load

Ingestion and automation

Automatically check all tracked channels on a recurring schedule

Only ingest videos published within the last 7 days

Avoid duplicates

If a video already exists, do not recreate it

If transcript or summary is missing, retry processing

Add basic logging for ingestion runs and failures

Make the scheduled sync easy to configure

Scope and UX

No login/auth needed unless required for admin protection

Personal dashboard only

Simple, clean, minimal UI

Responsive enough to work on laptop and mobile

Prefer practicality over fancy design

Recommended stack
Use a stack that is easy to build and maintain inside Replit:

Frontend: Next.js

Backend/API: Next.js API routes or a lightweight server inside the same project

Database: Supabase Postgres

Scheduled jobs: Replit scheduled jobs / cron

AI summaries: OpenAI API

Transcript fallback: speech-to-text pipeline

Styling: Tailwind CSS

Deployment: Replit deployable app

If you think a different stack is materially better inside Replit, explain why briefly in comments and proceed with the best pragmatic option.
